What is it?

Red Remote Protocol is an open protocol developed by Felipe Durar that provides remote access to a computer with Graphical Interface over the network sharing basic human interface devices such as mouse and keyboard and providing file transfer.

It is layered on top of the WebSocket protocol since it provides full-duplex communication channels over TCP with minimal framing, the data sent through the websocket are json encoded payload data as it will be described later.

More information about the WebSocket can be found at the RFC6455 provided by the Internet Engineering Task Force at https://datatracker.ietf.org/doc/html/rfc6455


Objectives

  • Be as simple as possible: The design philosophy of the protocol is that it could be simple enough to use basic features from other languages and even browsers to create client applications without having to deal with complex streaming or networking algothms and protocols, it also would provide a much easier code maintenance and a shorter learning curve for who's never saw the code earlier and wants to create a remote access client.

Costs

  • Performance: In order to keep the protocol simplicity some drawbacks were accepted what can sacrifice the performance of client applications running this protocol.
    For example, instead of using more consolidated and popular video streaming protocols the RRP send a framesegment (will be discussed later) packets that just represents the changed screen area, it is basically an image and the rectangle describing where it should be rendered. The image is compressed using some picture format (such as JPEG or PNG), converted to base64 and them added into the json payload.

How does it work?

Red Remote protocol was designed to be used with a central server which intermediates the socket connections between the clients providing several features such as authentication control, payload data validation among other things.
